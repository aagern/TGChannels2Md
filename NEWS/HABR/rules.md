# Правила суммаризации @habr_com

Канал: `@habr_com` — Telegram-зеркало Хабра.
Тип: **KNOWLEDGE/TUTORIAL**.
Контент: длинные технические статьи на русском + ежечасные новостные дайджесты «Новости к этому часу» + рекламный шум.
Источник данных: `posts_habr_com_<start>_<end>.md` — **использовать файл с максимальным диапазоном дат**; при наличии нескольких файлов брать только наиболее широкий, остальные игнорировать.

---

## Топик-фильтр

Каждый пост оценивается на принадлежность к одному или нескольким топикам. Посты, прошедшие фильтр, получают **полное описание**. Посты вне фильтра — **однострочное упоминание** (см. раздел «Остальное»).

| Тег | Попадает |
|-----|----------|
| `[DevOps]` | CI/CD, GitLab, Docker, Kubernetes, деплой, инфра-автоматизация |
| `[Linux]` | ядро Linux, systemd, journalctl, дистрибутивы, shell, bash |
| `[Rust]` | язык Rust, cargo, безопасная память, async runtime |
| `[DIY]` | самодельные устройства, электроника, Arduino/ESP, 3D-печать, hardware-хаки |
| `[Networking]` | TCP/IP, VPN, WireGuard, DNS, firewall, SDR, CDN, сетевые протоколы |
| `[AI/LLM]` | LLM, агенты, RAG, fine-tuning, инференс, языковые модели, ML-инфраструктура |
| `[HR]` | онбординг, найм, ретроспективы команд, управление, психология работы |
| `[Security]` | CVE, уязвимости, криптография, аутентификация, пентест, утечки данных |
| `[Sysadmin]` | мониторинг, логи, бэкапы, конфигурация серверов, Ceph, observability |

Статья может иметь несколько тегов — перечислять все применимые.

## Правило 1: Шум — удалять полностью (без упоминания)

Следующие типы постов не включаются ни в какой раздел:

- **Реклама курсов Хабра**: «На нашей витрине можно выбрать курс», «витрина курсов», «посмотреть курсы»
- **Сезонные мотивационные посты**: «майские уже близко», «пора выдохнуть», «хороший момент посмотреть на работу со стороны»
- **HR-флешмобы и опросы**: «перешлите этот пост вашим эйчарам», «участвуйте в опросе», «поделитесь с коллегами»
- **Промо и кросс-промо**: посты с тегом `#промо`, посты с основным CTA «подпишитесь» / «подпишись»
- **Платные курсы / интенсивы / вебинары**: «курс», «обучение», «интенсив», «мастермайнд», «марафон» как основная тема с платным доступом
- **Записи на события**: основное действие — «записаться» / «зарегистрироваться» + ссылка на онлайн-мероприятие

## Правило 2: Классификация сигнала

Для постов, прошедших топик-фильтр:

### ACTIONABLE
Туториал с работающим кодом или конфигом, CVE с PoC, breaking change API, анонс инструмента с репо или ссылкой на скачивание.

### NOVEL
Принципиально новый подход, первый в своём роде инструмент, крупный релиз с реально новыми возможностями (не «улучшенный», а «раньше так нельзя было»).

### BUSINESS_CASE
Реальное внедрение: конкретный стек + конкретная решённая задача. Должно быть ясно ЧТО сделали и КАК.

### TIER 3 (на радаре)
Тренд-статья, обзор без кода, экосистемный сигнал, исследование без релиза.

---

## Правило 3: Уровни срочности

- **TIER 1 (Срочно)**: CVE / security patch / breaking API change / крупный сервис лёг — нужно действовать сегодня
- **TIER 2 (На этой неделе)**: ACTIONABLE или NOVEL по любому из топиков — стоит изучить в ближайшие дни
- **TIER 3 (На радаре)**: интересно, но не срочно

BUSINESS_CASE идут в свой раздел, не в тиры.

## Правило 5: «Новости к этому часу» — специальная обработка

Распознаётся по: заголовок «⚡ Новости к этому часу» и/или тег `#новости`.

**Правила:**
1. Включать **все пункты** дайджеста — топик-фильтр к дайджестам не применяется.
2. Строку «ПОДДЕРЖИТЕ КАНАЛ ГОЛОСАМИ» — пропустить.
3. Каждый пункт — одна строка: короткая фраза + ссылка на habr.com.
4. Весь дайджест выводится одним блоком в разделе `📰 Дайджесты`.

**Формат блока:**
```
### 📰 Дайджест ДД.ММ.ГГГГ ЧЧ:ММ | [TG](https://t.me/habr_com/XXXXX)
• Краткое описание новости — [ссылка](https://habr.com/...)
• Краткое описание новости — [ссылка](https://habr.com/...)
...
```

## Правило 6: Посты вне топик-фильтра — раздел «Остальное»

Посты, не прошедшие топик-фильтр (ретрогейминг, музыкальная история, астрономия, геймдизайн, вакуумные лампы и т.п.), выводятся одной строкой без описания:

```
• Заголовок статьи — [Habr](https://habr.com/...)
```

Собираются в раздел `📦 Остальное` в конце вывода.

---

## Правило 7: Флаги скептицизма

Добавлять к записям, где применимо:

- `[NO-CODE]` — нет репозитория или реализации
- `[SELF-REPORTED]` — цифры не верифицированы независимо
- `[VENDOR]` — материал от компании, продающей описываемый продукт

При наличии 2+ флагов — свернуть до заголовка + флаги, без полного описания.

---

## Правило 8: Советы из туториалов

Извлекать из туториалов конкретные команды, флаги, конфиги, паттерны, предупреждения о подводных камнях.
Формат — максимум 2 строки на совет, без ссылок, актуально независимо от даты.
